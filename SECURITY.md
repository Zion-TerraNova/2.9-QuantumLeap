# Security Policy

## Reporting a vulnerability
Please report security issues privately.

- Preferred: open a private security advisory in GitHub Security tab (if enabled)
- Or email: security@your-domain.example (replace)

Do not open public issues for vulnerabilities.

## Scope
This repository must never contain:
- API keys / secrets (Stripe, wallet APIs, etc.)
- Private keys, seed phrases, mnemonics
- Production hostnames/IPs, SSH credentials
